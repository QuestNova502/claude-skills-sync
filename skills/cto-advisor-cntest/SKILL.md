# CTO 顾问（中文测试版）

技术领导力的战略框架和工具，用于团队扩展和工程卓越。

## 关键词
CTO, 首席技术官, 技术领导力, 技术债务, 工程团队, 团队扩展, 架构决策, 技术评估, 工程指标, DORA 指标, ADR, 架构决策记录, 技术战略, 工程领导力, 工程组织, 团队结构, 招聘计划, 技术策略, 供应商评估, 技术选型

## 快速开始

### 技术债务评估
```bash
python scripts/tech_debt_analyzer.py
```
分析系统架构并提供优先级债务削减计划。

### 团队扩展规划
```bash
python scripts/team_scaling_calculator.py
```
计算最优招聘计划和团队结构以支持增长。

### 架构决策
查看 `references/architecture_decision_records.md` 获取 ADR 模板和示例。

### 技术评估
使用 `references/technology_evaluation_framework.md` 中的框架进行供应商选型。

### 工程指标
从 `references/engineering_metrics.md` 实施 KPI 以跟踪团队表现。

## 核心职责

### 1. 技术战略

#### 愿景与路线图
- 定义 3-5 年技术愿景
- 创建季度路线图
- 与业务战略对齐
- 向利益相关者传达

#### 创新管理
- 分配 20% 时间用于创新
- 每季度举办黑客马拉松
- 评估新兴技术
- 构建概念验证

#### 技术债务策略
```bash
# 评估当前债务
python scripts/tech_debt_analyzer.py

# 分配资源容量
- 关键债务：40% 容量
- 高优先级债务：25% 容量
- 中优先级债务：15% 容量
- 低优先级债务：日常维护
```

### 2. 团队领导力

#### 工程团队扩展
```bash
# 计算扩展需求
python scripts/team_scaling_calculator.py

# 关键比例维护：
- 管理者:工程师 = 1:8
- 高级:中级:初级 = 3:4:2
- 产品:工程 = 1:10
- QA:工程 = 1.5:10
```

#### 绩效管理
- 每季度设定明确的 OKR
- 每周进行 1:1 会谈
- 每季度审查绩效
- 提供成长机会

#### 文化建设
- 定义工程价值观
- 建立编码标准
- 创建学习项目
- 促进协作

### 3. 架构治理

#### 决策制定
使用 `references/architecture_decision_records.md` 中的 ADR 模板：
1. 记录背景和问题
2. 列出所有考虑的选项
3. 记录决策和理由
4. 跟踪后果

#### 技术标准
- 语言选择
- 框架选型
- 数据库标准
- 安全要求
- API 设计指南

#### 系统设计评审
- 每周架构评审
- 设计文档标准
- 原型要求
- 性能标准

### 4. 供应商管理

#### 评估流程
遵循 `references/technology_evaluation_framework.md` 中的框架：
1. 收集需求（第 1 周）
2. 市场调研（第 1-2 周）
3. 深度评估（第 2-4 周）
4. 决策和文档化（第 4 周）

#### 供应商关系
- 季度业务评审
- SLA 监控
- 成本优化
- 战略合作伙伴关系

### 5. 工程卓越

#### 指标实施
来自 `references/engineering_metrics.md`：

**DORA 指标**（部署到生产环境的目标）：
- 部署频率：>1 次/天
- 交付周期：<1 天
- 平均恢复时间（MTTR）：<1 小时
- 变更失败率：<15%

**质量指标**：
- 测试覆盖率：>80%
- 代码评审：100%
- 技术债务：<10%

**团队健康度**：
- Sprint 速率：±10% 波动
- 计划外工作：<20%
- On-call 事件：<5 次/周

## 危机管理

### 事件响应

1. **立即响应**（0-15 分钟）：
   - 评估严重性
   - 激活事件团队
   - 开始沟通

2. **短期处理**（15-60 分钟）：
   - 实施修复
   - 更新利益相关者
   - 监控系统

3. **问题解决**（1-24 小时）：
   - 验证修复
   - 记录时间线
   - 客户沟通

4. **事后总结**（48-72 小时）：
   - 根因分析
   - 行动项
   - 流程改进

### 危机类型

#### 安全漏洞
- 隔离受影响系统
- 启动安全团队
- 法律/合规通知
- 客户沟通计划

#### 重大故障
- 全员响应
- 状态页面更新
- 高管简报
- 客户外联

#### 数据丢失
- 立即停止写入
- 评估恢复选项
- 开始恢复
- 影响分析

## 成功指标

✅ **技术卓越**
- 系统正常运行时间 >99.9%
- 每天多次部署
- 技术债务 <10% 容量
- 安全事件 = 0

✅ **团队成功**
- 团队满意度 >8/10
- 流失率 <10%
- 职位填补率 >90%
- 多样性提升

✅ **业务影响**
- 功能按时交付 >80%
- 工程赋能收入
- 单笔交易成本下降
- 创新驱动增长
