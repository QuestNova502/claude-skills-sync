# 周度产业研究报告生成器

根据用户指定的主题/行业，通过网络调研收集最新数据，生成结构化的专业产业研究报告，输出为 .docx 格式。

## 触发条件

当用户提到以下内容时自动触发：
- "生成报告"、"写报告"、"产业报告"、"行业报告"
- "每周报告"、"周度报告"、"weekly report"
- "趋势分析"、"产业研究"、"行业研究"
- "生成一份关于 XXX 的报告"

## 报告结构模板

报告必须严格遵循以下结构：

```
[报告标题]：[副标题——趋势、形态与战略应对]

------面向[目标读者]的产业研究报告

报告编号：[领域缩写]-[子领域]-[年份]-[序号]
发布日期：[年] 年 [月] 月
研究机构：[机构名称或用户指定]
报告级别：VP-Level Deliverable
保密等级：Client Confidential

─── 执行摘要（Executive Summary）───
- 背景概述（2-3句）
- 本报告的研究基础和数据来源概述
- 三个核心判断，每个判断包含：
  - 加粗标题
  - 关键数据引用（带编号引用标记 [N]）
  - 趋势判断
- 战略建议总结

─── 第一章 研究背景与方法 ───
1.1 研究背景
  - 宏观趋势描述
  - 多项权威数据引用（附数据表格）
1.2 研究核心问题
  - 3个核心研究问题（加粗）
1.3 研究方法
  - (1) 产业数据分析
  - (2) 学术研究分析
  - (3) 行业案例分析
1.4 数据来源
  - 数据来源汇总表格

─── 第二章 [主题]的系统性影响 ───
2.1 [维度一]
  - 关键结论（灰底高亮框）
  - 数据与分析（多段落+引用）
  - 数据汇总表格
  - 影响分析（3个结构性变化，加粗标题）
2.2 [维度二]
  - 同上结构
2.3 [维度三]
  - 同上结构

─── 第三章 未来形态预演 ───
3.1 形态一：[名称]
  - 核心特征
  - 产品形态描述
  - 对比表格（传统 vs 新模式）
  - 影响分析
3.2 形态二：[名称]
  - 同上结构

─── 第四章 平台/企业应对策略 ───
- 战略判断（加粗核心结论）
- 关键能力表格（能力维度 | 能力描述 | 竞争意义）
- 3个建议战略方向（加粗标题+详细描述）

─── 第五章 风险与不确定性 ───
5.1 风险一
5.2 风险二
5.3 风险三
（每个风险包含数据支撑和分析）

─── 第六章 核心结论 ───
- 判断一/二/三（加粗核心观点）
- 战略建议总结表格（战略方向 | 核心目标 | 关键行动 | 时间窗口）

─── 参考文献与数据来源 ───
[1] 至 [N]，每条包含：作者/机构, 标题, 出版物, 日期, URL

─── 免责声明 + 版权声明 ───
```

## 执行流程

### 步骤 1：确认主题与参数

向用户确认：
- **报告主题**：例如"生成式AI对社交媒体的影响"
- **目标读者**：例如"社交媒体平台决策层"
- **研究维度**：默认3个维度，用户可自定义
- **报告编号前缀**：例如 TMT-AI
- **研究机构名称**：默认"Manus Strategy & Technology Research"
- **输出路径**：默认桌面

### 步骤 2：网络调研

使用 WebSearch 工具进行多轮搜索，收集：
1. **权威产业数据**：来自 Stanford HAI、McKinsey、Gartner、Counterpoint Research、IDC 等
2. **学术研究**：来自 Nature、arXiv、Science 等
3. **行业报告**：来自 HubSpot、Hootsuite、Sprout Social 等
4. **新闻与案例**：来自 BBC、NYT、Forbes、TechCrunch 等
5. **市场数据**：具体数字、增长率、市场份额

**搜索策略**：
- 围绕主题的每个维度分别搜索
- 优先搜索最近6个月的数据
- 每个关键观点至少有2个独立数据源交叉验证
- 收集至少 15-25 条可引用的参考文献

### 步骤 3：生成报告内容

根据收集的数据，按照报告结构模板生成完整内容：

**写作要求**：
- 语言风格：专业、严谨、战略视角
- 数据引用：所有数据必须标注引用编号 [N]
- 表格：每章至少包含1个数据表格
- 关键结论：使用灰底框突出显示
- 加粗：核心判断和重要数据使用加粗
- 篇幅：总字数 8000-15000 字（中文）

### 步骤 4：生成 .docx 文件

使用 docx skill 的能力生成格式化的 Word 文档：

1. 先读取 docx skill 的 `docx-js.md` 文档了解 API
2. 编写 JavaScript 脚本生成 .docx 文件
3. 文档格式要求：
   - 标题：黑体/微软雅黑，24pt
   - 一级标题：20pt，加粗
   - 二级标题：16pt，加粗
   - 正文：12pt，宋体/等线
   - 表格：带边框，表头灰底加粗
   - 页边距：上下2.54cm，左右3.17cm
   - 行距：1.5倍
   - 引用编号：上标格式

### 步骤 5：输出与确认

1. 将 .docx 文件保存到用户指定路径
2. 告知用户报告概要（标题、章节数、引用数、字数）
3. 询问是否需要调整

## 报告质量标准

1. **数据准确性**：所有数据必须来自可验证的公开来源
2. **引用完整性**：每个数据点必须有对应的参考文献
3. **逻辑严密性**：从数据到结论的推理链条完整
4. **战略价值**：结论必须具有可操作的战略指导意义
5. **格式一致性**：严格遵循模板结构，不遗漏任何部分

## 快速启动示例

用户输入：
```
帮我生成一份关于"AI Agent对企业SaaS市场的冲击"的周度产业研究报告
```

AI 执行：
1. 确认主题参数
2. 搜索 AI Agent + SaaS 相关的最新数据（至少5轮搜索）
3. 从生产侧、消费侧、竞争格局三个维度组织内容
4. 生成完整报告结构
5. 输出为 .docx 文件到桌面

## 周度报告序号管理

报告编号格式：`[领域]-[子领域]-[年份]-[序号]`

示例序列：
- TMT-AI-2026-001
- TMT-AI-2026-002
- TMT-AI-2026-003

每次生成新报告时，检查输出目录中已有的报告编号，自动递增序号。

## 注意事项

- 搜索时使用当前年份（2026年）确保数据时效性
- 如果某个数据点无法找到可靠来源，应明确标注"据行业估算"而非捏造
- 表格数据必须与正文描述一致
- 参考文献的 URL 必须是搜索中实际找到的链接
- 报告生成后建议用户用 Word 打开检查格式
