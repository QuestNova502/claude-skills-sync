# Claude Skills GitHub同步方案

## 🎯 设计目标

1. **最小权限原则** - 只同步必要的内容，保护隐私
2. **跨设备同步** - 方便在不同电脑间使用
3. **易于分享** - 可以安全地和朋友共享
4. **未来扩展** - 为其他项目预留空间
5. **简单维护** - 自动化同步流程

---

## 📁 推荐的仓库结构

### 方案A: 单独的Skills仓库（推荐）⭐

```
claude-skills-backup/
├── README.md                    # 使用说明
├── .gitignore                   # 忽略规则
├── sync.sh                      # 同步脚本
├── install.sh                   # 安装脚本
├── skills/                      # skills备份
│   ├── product-manager-prompts/ # 你安装的skills
│   ├── competitive-ads-extractor/
│   ├── jira-integration/
│   └── ... (所有153个skills)
├── docs/                        # 你的文档
│   ├── Skills介绍.md
│   ├── 来源分析.md
│   └── 使用指南.md
└── scripts/                     # 实用脚本
    ├── list-skills.sh           # 列出所有skills
    └── update-from-github.sh    # 从GitHub更新
```

**优点**:
- 职责单一，专门管理skills
- 方便分享给朋友（整个仓库克隆即可）
- 未来其他项目不受影响
- 可以设置为Private仓库

**缺点**:
- 需要单独维护一个仓库

### 方案B: Monorepo（多项目单仓库）

```
my-workspace/
```

**优点**:

## 🚀 实施步骤

### 步骤1: 创建GitHub仓库

```bash
# 在GitHub网页端创建私有仓库
仓库名: claude-skills-sync
可见性: Private
描述: My Claude Code Skills backup and sync
不要初始化README（我们本地创建）
```

### 步骤2: 本地初始化

```bash
# 创建项目目录
mkdir ~/claude-skills-sync
cd ~/claude-skills-sync

# 初始化Git
git init
git branch -M main

# 创建基础结构
mkdir -p skills docs scripts
```

### 步骤3: 复制skills

```bash
# 复制所有skills到仓库
cp -r ~/.claude/skills/* ./skills/

# 复制你的文档
cp ~/Desktop/Claude_Skills_介绍.md ./docs/
cp ~/Desktop/Skills资源详细对比.md ./docs/
cp ~/Desktop/产品经理Prompts使用指南.md ./docs/
```

### 步骤4: 创建.gitignore

```bash
# 创建忽略规则文件
cat > .gitignore << 'EOF'
# 见上面的忽略规则
EOF
```

### 步骤5: 推送到GitHub

```bash
# 添加远程仓库
git remote add origin https://github.com/你的用户名/claude-skills-sync.git

# 首次提交
git add .
git commit -m "Initial commit: Add 153 Claude skills and documentation"
git push -u origin main
```

## 🔄 日常同步工作流

### 自动同步脚本（sync.sh）

```bash
#!/bin/bash
# 自动同步脚本

# 从本地skills目录同步到仓库
rsync -av --delete ~/.claude/skills/ ~/claude-skills-sync/skills/

# 自动提交
cd ~/claude-skills-sync
git add .
git commit -m "Sync: $(date '+%Y-%m-%d %H:%M')"
git push origin main

echo "✅ Skills已同步到GitHub"
```

### 从GitHub更新到本地（install.sh）

```bash
#!/bin/bash
# 在新电脑上安装skills

# 从GitHub克隆
git clone https://github.com/你的用户名/claude-skills-sync.git
cd claude-skills-sync

# 复制到Claude目录
mkdir -p ~/.claude/skills
cp -r ./skills/* ~/.claude/skills/

echo "✅ Skills已安装到本地"
```

## 👥 与朋友共享

### 方式1: 邀请协作者（推荐）

1. GitHub仓库 → Settings → Collaborators
2. 输入朋友的GitHub用户名
3. 发送邀请
4. 朋友接受后可以访问Private仓库

### 方式2: 生成访问令牌（只读）

1. GitHub → Settings → Developer settings → Personal access tokens
2. 生成只读令牌
3. 分享给朋友，他们可以克隆但不能修改

### 方式3: 导出为压缩包

```bash
# 定期导出
cd ~/claude-skills-sync
tar -czf claude-skills-$(date +%Y%m%d).tar.gz skills/ docs/
# 通过其他方式分享给朋友
```

## 💡 高级特性

### 使用Git Submodules（可选）

如果某些skills来自其他仓库，可以用submodule：

```bash
# 添加原始仓库作为submodule
git submodule add https://github.com/anthropics/skills.git skills/anthropics-official
git submodule add https://github.com/alirezarezvani/claude-skills.git skills/alireza-skills
```

### 使用GitHub Actions自动化（可选）

定期自动备份和同步：

```yaml
# .github/workflows/sync.yml
name: Auto Sync Skills

on:
  schedule:
    - cron: '0 0 * * 0'  # 每周日自动运行
  workflow_dispatch:      # 手动触发

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Sync skills
        run: |
          echo "Skills synced at $(date)"
```

## ⚠️ 注意事项

### 安全检查清单

在首次推送前检查：

- [ ] 没有包含API keys
- [ ] 没有包含密码
- [ ] 没有包含个人邮箱/手机
- [ ] 没有包含公司内部代码
- [ ] 没有包含客户数据
- [ ] 仓库设置为Private

### 文件大小限制

- GitHub单文件限制: 100MB
- 仓库推荐大小: < 1GB
- 如果skills包含大文件，考虑用Git LFS

### License考虑

你的仓库应该包含LICENSE文件：
- 对于你自己的内容: MIT License
- 对于复制的skills: 保留原始license
